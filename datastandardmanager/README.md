## 特性
###v1.7.0
- 数据探查增加增加接入方式信息配置项,将修改后的信息推送至定义模块（调用这边的保存接口）
- 数据定义详情页面增加新的接入方式信息展现
- 新增物理表存储表 OBJECT_STORE_INFO ，建表、修改表结构时保存物理表相关信息
- 数据定义管理页面 新增已建物理表的查询参数  增加按钮“刷新已建表” 点击按钮批量刷新每个标准在平台中建表信息（离线库&在线库），
  将最新建表信息更新到标准库表synlte. BJECT_STORE_INFO中
###v1.6.0
- 数据组织页面可以跳转到标准管理页面，通过参数tableId自动回填相关参数（未做）
- 从数据探查页面跳转到数据定义页面，探查字段元素编码自动回填元素集列，包括名称、类型、长度信息。
- 字段列表增加“分类”列，分类信息通过元素集关联回填，不可编辑（依赖标委会给出分级定义）。
- 字段列表“敏感等级”列，通过设置元素集后，通过对应语义关联回填敏感级（1~99）信息，可编辑。
- 标准保存时同时保持object、objectfield、PUBLIC_DATA_INFO、FIELD_RESOURCE_INFO表。
- 已建表单独在新的标签页中展示，不在建表页面中展示
- 支持dataHub建表  需设置通道数，默认50  Dt字段必须存在，且是最后一个字段 可选择工程名称
   不支持修改表结构，只能删除重建
-  修改标准，增加新字段定义，同步更新表结构 Odps\hive\hbase、ads实时表，可增加支持alter table; 
    Ads、dataHub批量表不支持alter table;
- 如果dataid不存在 则使用数据仓库中的本地数据库
- Object表增加4个字段 , CREATETIME：标准新增时间，默认sysdate  ,UPDATETIME：标准最新修改时间，
   保存定义编辑信息时的服务器时间。CREATOR：标准创用户名称，新建定义时登录用户信息，
   第三方调用页面时保存“源应用系统建设公司”信息。 UPDATER：标准最新修改用户名称，编辑操作时登录用户信息，
   第三方调用页面时保存“源应用系统建设公司”信息。
-  数据定义页面增加新属性，展现创建时间、创建用户、最新修改时间、最新修改用户信息。
-  左侧分类目录，增加排序类型设置功能，按照名称排序和创建时间两种方式切换。

###v1.5.0
- 新增数据历程功能，当在大数据平台建表成功时，使用aop将建表信息发送到数据历程的接口
- 适配达梦和金仓国产数据库，配置里面数据库的连接信息发生变化
- 数据仓库中新增了相似度分析，当从数据仓库获取字段信息时要把相似度分析的结果(字段探查出来的元素编码)
  回填到编辑页面中，其中表Dsm_Source_Field_Info新增字段fieldCode，用来存储分析后的元素编码。
