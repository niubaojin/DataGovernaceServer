#数据产品线-数据血缘

## 特性
###v1.7.0
- 引入hazelcast 做分布式缓存和分布式锁，为以后部署多个程序做准备
- 如果需要分布式部署，部署多个程序，则需要等其中一个程序运行结束后再运行其它程序，否则无法相互注册
###v1.6.0
- 应用环节数据进行血缘解析关联时的，只使用表名，忽略项目名或平台信息，达到通用效果。-、数据加工血缘解析时，过滤掉生产库(项目名_dev结尾)的数据记录，简化加工血缘数据关系。
-  数据处理血缘，从标准库中的数据提取规则映射表中获取 可借鉴李龙现成的代码来解析相关的数据（与数据处理迁移同步完成）。
- 从数据组织页面可以跳转到数血缘页面，定位到数据加工环节，以当前表名（离线库）为中心点的组织加工血缘关系。
- syndmg_table_all这张表项目名更改为 synlte 
- 数据处理血缘不再调用李龙的接口，直接把代码迁移到这个程序中
